Deactivate a Doctor
by ID

Executes a PostgreSQL UPDATE that sets a doctor's IsActive status to false. Validates the ID exists first, confirms row count after update, and reports whether the operation succeeded.

⚙️ Execute Procedure
Enter the Doctor's numeric ID to deactivate their account. This mirrors calling CALL deactivate_doctor(id).

How the Procedure Works
1
ID Lookup — Checks if the Doctor ID exists in the database before proceeding.
2
UPDATE — Sets IsActive = false on the matching doctor record.
3
Row Count Check — Validates rows_affected > 0 to confirm the operation succeeded.
4
Feedback — Returns a success message with the doctor's name, or a not-found error.
